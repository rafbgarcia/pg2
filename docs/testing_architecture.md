# Testing Architecture

This document defines the long-term test lane model for `pg2`.
The objective is strict separation by contract level first, then runtime cost.

## Test Lanes

1. `unit`
- Scope: local component logic and invariants.
- Dependency rule: no session-path integration unless explicitly required by contract.
- Runtime target: fastest lane.
- Build targets:
  - `zig build unit` (runs inline `src/*` unit tests).

2. `features`
- Scope: user-facing behavior contracts (1:1 supported product features).
- Dependency rule: must use the feature harness; no direct internal mutation shortcuts.
- Location: `test/features`.

3. `internals`
- Scope: non-user-facing engine contracts (durability, reclaim, spill, protocol internals).
- Dependency rule: must not import feature wrapper paths.
- Location: `test/internals`.

4. `sim`
- Scope: deterministic fault-schedule replay and crash/recovery matrices.
- Dependency rule: deterministic seeds and deterministic signatures required.
- Location: `test/sim`.

5. `stress`
- Scope: heavy time/resource gates (RSS, throughput, long workloads).
- Dependency rule: excluded from default fast lane.
- Location: `test/stress`.

## Harness Ownership

- `test/harness/feature_env.zig`: session-path setup for user-facing and stress-style end-to-end tests.
- `test/harness/internal_env.zig`: internal lane boundary; internal tests import this path.
- `test/shared/*`: compatibility layer only while migration is in progress.

## Build Targets

- `zig build test`: fast lane (`features`, `internals`) plus deterministic simulation gate.
- `zig build unit`: run inline `src/*` unit tests.
- `zig build sim`: simulation lane only.
- `zig build stress`: heavy stress lane only.

## Enforcement Rules

- Suite roots are generated by `scripts/generate_test_suites.sh`.
- `test/test_discovery_guard_test.zig` enforces:
  - every managed test is included exactly once in a suite root,
  - internal tests do not import feature wrapper paths.

## Migration Notes

- Inline `src/*` tests are the primary unit lane while module-private invariants remain colocated.
- `test/unit/*` is optional and can host cross-module black-box unit scenarios when needed.
